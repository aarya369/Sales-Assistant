## Fairness

The system may produce slightly different SQL queries depending on how a question is phrased. Natural language models are sensitive to wording, abbreviations, and regional variations of English. To evaluate this, equivalent questions such as “Top selling products”, “Which products sold the most?”, and “What are the best-selling products?” were tested. The generated SQL was functionally similar in these cases. However, variations in phrasing could still affect query quality, particularly for ambiguous business terms. Human review and query validation help reduce this risk.
## Transparency

Users interact with an AI-powered sales assistant that converts natural language questions into SQL queries. The system displays the generated SQL before execution, allowing users to inspect and validate the query. This increases transparency and helps users understand how answers are produced. Users should be informed that the responses are generated by an AI system and may occasionally contain errors or incorrect assumptions.
## Accountability

If the system returns an incorrect answer, the failure may originate from multiple stages including misunderstanding the user's question, generating incorrect SQL, retrieving incomplete data, or presenting misleading results. To improve accountability, the generated SQL is displayed before execution and all queries can be logged for auditing purposes. Query validation, guardrails, and result sanitization reduce risk, but human oversight remains important for high-impact business decisions.
## Data Minimisation

The system is designed to retrieve only the data required to answer a user's question. Relevant schema information and allowed values are selected dynamically rather than exposing the full database schema to the model. SQL queries are restricted to read-only operations and result sets are automatically bounded using LIMIT clauses. Result sanitization removes unnecessary or sensitive fields before displaying output. These measures help reduce unnecessary data exposure and support the principle of data minimisation.
## Conclusion

The Sales Insights Assistant incorporates multiple responsible AI controls including input guardrails, SQL validation, output guardrails, result sanitization, and data minimisation measures. While these controls improve safety and reliability, the system should be viewed as a decision-support tool rather than a replacement for human judgment. Critical business decisions should continue to involve human review and validation.
